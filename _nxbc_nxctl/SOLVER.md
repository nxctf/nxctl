# 04 Convergence Solver

This solver is the launcher-aware version of the original `_tmp/convergence/solver.py`.

In launcher mode, the player does not create or fund a wallet manually. The portal returns:

```text
RPC_URL
PRIVKEY
SETUP_ADDR
```

Then the solver uses that temporary CTF wallet to solve the per-user setup instance.

## Recommended Flow

Start the POC:

```sh
docker compose down
docker compose up --build
```

Run the full launcher test:

```sh
RESET_PORTAL_STATE=1 python3 scripts/test_portal_flow.py
```

## Manual Run

Launch first:

```sh
curl -s -X POST http://localhost:8080/launch/04-convergence \
  -H "X-User-Id: user-a"
```

Export values from the launch response:

```sh
export RPC_URL=http://localhost:8545
export PRIVKEY=0x...
export SETUP_ADDR=0x...
python3 solver.py
```

Expected ending:

```text
solved:   True
```

Then request the flag:

```sh
curl -s -X POST http://localhost:8080/check/04-convergence \
  -H "X-User-Id: user-a"
```

## Notes

- `PRIVKEY` is a disposable CTF wallet generated by the launcher.
- Do not use real wallets here.
- The flag is not on-chain.
- The checker uses the setup instance stored for the logged-in user.
